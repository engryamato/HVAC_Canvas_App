# Execute

1. Create or update `docs/SETUP.md` with exact setup/start/test commands.
2. Create or update `docs/ARCHITECTURE.md` with service overview and key request flows.
3. Create or update `docs/RUNBOOK.md` with operations, logs, migrations, seeds, and reset flow.
4. Create or update `docs/DECISIONS.md` with concise ADR-style entries.
5. Ensure all docs reference existing scripts only.

# Quality Rules

- Keep instructions copy/paste ready.
- Include troubleshooting for common failures.
- Keep decision entries dated and rationale-driven.

# Inputs

- Verified commands and scripts
- Service boundaries and dependencies
- Key implementation choices

# Outputs

- Operator-friendly documentation set
- Auditable decision trail
